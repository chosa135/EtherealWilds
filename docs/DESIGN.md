# DESIGN

この文書は、Codexなどのコーディングエージェントが仕様を誤解せずに実装を進めるための設計メモです。

## 企画の核

Ethereal Wilds Prototype は、FE風のターン制グリッドSRPGをベースに、ローグライク的なラン進行とリソース管理を組み合わせる実験作です。

重視している体験は以下です。

- 小さなマップをテンポよく連戦する。
- ラン中の報酬で武器や消費アイテムを補強する。
- 武器耐久、強撃、修繕、所持品枠の制約で判断を作る。
- ランダム成長と能力強化アイテムで、同じユニットでもランごとに少し違う育ち方をする。

## 現在のゲームループ

```txt
ワールドマップ
  ↓
戦闘マップ
  ↓ 敵全滅
戦闘報酬 3択
  ↓ 報酬を誰かに持たせる、または受け取らずに進む
ワールドマップ
  ↓
休憩所ノードなら休憩所
  ↓
次の戦闘へ
```

ワールドマップは現在一本道です。分岐、イベント、宝、店などは未実装です。

## 戦闘の基本仕様

### HP0

HP0は死亡ではなく「戦闘不能」です。

- 自軍ユニットがHP0になると `unavailable = true` になります。
- そのマップから撤退し、行動済みになります。
- 休憩所の「復帰」で1人を復帰できます。
- 永続ロストはありません。

### 反撃と追撃

- 攻撃後、相手が生存していて射程内なら反撃します。
- 速さ差4以上で追撃します。
- 攻撃側だけでなく、反撃側も速ければ追撃反撃します。
- 攻撃順は以下です。

```txt
攻撃側の初撃
  ↓
防御側の反撃
  ↓
防御側が速い場合、追撃反撃
  ↓
攻撃側が速い場合、追撃
```

### 命中

表示命中は以下です。

```txt
命中 = 武器命中 + 攻撃側の技 * 2 - 防御側の速さ * 2
```

判定は2RNです。

### 強撃

強撃は、武器耐久を多めに消費する強力な攻撃です。

- 1マップ1回。
- ターン終了では回復しません。
- マップ開始時に回復します。
- 強撃扱いになるのは初撃だけです。
- 追撃が発生した場合、追撃は通常攻撃です。
- 強撃の耐久消費は3です。
- 通常攻撃の耐久消費は1です。

強撃ダメージは以下です。

```txt
強撃ダメージ = 通常ダメージ + 3 + max(攻撃側の技 - 防御側の技, 0)
```

### 挟撃

挟撃システムは一旦取り除いています。再導入する場合は、戦闘計算が複雑になりすぎない形にしてください。

## 移動仕様

同陣営ユニットのマスは、移動中は通過可能ですが停止不可です。

- 味方は味方を通過できます。
- 敵は敵を通過できます。
- 敵対陣営ユニットのマスは通過不可です。
- 壁は通過不可です。
- 森は移動コスト2です。

## 所持品・装備

各ユニットは所持品4枠を持ちます。

- 武器と消費アイテムは同じ所持品欄で管理します。
- 装備中武器は `equippedItemId` で管理します。
- 攻撃、強撃、命中、耐久消費は装備中武器を参照します。
- 装備変更は行動権を消費しません。
- 道具使用は行動権を消費します。
- 空き枠がない場合、戦闘報酬はそのユニットに渡せません。

所持品関連の処理は `src/logic/inventory.ts` に置きます。

## 武器ランクと役割

現在の武器設計は以下です。

| ランク | 例 | 役割 |
| --- | --- | --- |
| 初期装備 | 青銅 | 低威力・高耐久。詰み防止用。 |
| コモン | 鉄 | 通常運用の主力。 |
| アンコモン | 白磁、古びた魔導書 | 高威力・低耐久。切り札。 |
| レア | 白銀 | 高性能・中耐久。ラン中の当たり武器。 |

初期武器だけで楽にクリアできる状態は避けたい方針です。青銅系は最後の保険であり、主力武器は報酬で得る想定です。

## 消費アイテム

消費アイテムには、回復アイテムと能力強化アイテムがあります。

- 傷薬: HP回復。
- 上傷薬: HP大回復。
- 雫: 対応能力+1。
- 大雫: 対応能力+3。

能力強化対象は、力、魔力、技、速さ、守備、魔防です。最大HP強化は現在未実装です。

## 戦闘後報酬

戦闘後に3つの候補が提示され、1つだけ選べます。

各候補は独立抽選です。

```txt
1. レアリティ抽選
   コモン 70%
   アンコモン 25%
   レア 5%

2. カテゴリ抽選
   武器 60%
   消費アイテム 40%

3. 対応する報酬テーブルから抽選
```

カテゴリ保証はありません。そのため、候補が3つとも武器、または3つとも消費アイテムになることがあります。これは意図した仕様です。

報酬関連のデータは `src/data/rewards.ts`、抽選処理は `src/logic/rewards.ts` に置きます。

## 休憩所

休憩所では2回行動できます。

- 休息: 出撃可能な味方全員のHPを最大HPの50%ぶん回復。
- 復帰: 戦闘不能者1人を復帰。HPは最大HPの50%。
- 鍛錬: 出撃可能な味方全員にEXP+30。
- 修繕: 味方の全所持武器から1つ選び、耐久を最大値の50%ぶん回復。

マップ終了時の自動回復はありません。

## 成長

レベルアップ時は以下です。

- HP+1は固定。
- 力、魔力、技、速さ、守備、魔防から、重み付きランダムで合計3ポイント上昇。
- 同じ能力に複数ポイント入ることがあります。

成長処理は現在 `src/main.ts` にあります。今後膨らむ場合は `src/logic/growth.ts` への分離候補です。

## データ配置方針

マスターデータは `src/data/` に置きます。

- `weapons.ts`: 武器マスター。
- `items.ts`: 消費アイテムマスター。
- `playerUnits.ts`: 味方初期値。
- `enemies.ts`: 敵基本値。
- `maps.ts`: マップ、敵配置、ワールドノード。
- `rewards.ts`: 報酬レート、報酬テーブル。

ラン中に変化する実体データは、`src/logic/factories.ts` でマスターデータから生成します。マスターデータに現在耐久や現在HPのような可変値を持たせないでください。

## コード配置方針

- 戦闘計算は `src/logic/combat.ts`。
- 敵AIは `src/logic/enemyAI.ts`。
- 所持品・装備処理は `src/logic/inventory.ts`。
- 報酬抽選は `src/logic/rewards.ts`。
- データから実体を作る処理は `src/logic/factories.ts`。
- `main.ts` はゲーム状態、進行、入力、描画の接着に寄せます。

新機能を追加するときは、まずどの責務に属するかを考え、`main.ts` に巨大な処理を足さないでください。
