# Ethereal Wilds Prototype

FE風のグリッドSRPGに、ローグライク的なラン進行、武器耐久、戦闘後報酬、休憩所でのリソース管理を組み合わせたブラウザ用プロトタイプです。実装は TypeScript + Vite + Canvas です。

UIは、禁足の樹海を探索する落ち着いた中世ファンタジーを基調とし、暗い木目・金属色のウインドウ、金のアクセント、青い選択表示、赤い攻撃表示で構成しています。

このリポジトリは Codex などのコーディングエージェントへ移行しやすいように、仕様・TODO・変更履歴を `docs/` に分けています。実装作業の前には、まず `docs/DESIGN.md` と `docs/CODEX_GUIDE.md` を読んでください。

## セットアップ

```bash
npm install
npm run dev
```

ブラウザで表示されたローカルURLを開いてください。

ビルド確認は以下です。

```bash
npm run build
```

## 現在のゲーム内容

- 8x6の小型戦闘マップを連続で進む、短いラン形式のSRPG。
- ワールドマップは一本道。戦闘、休憩所、終点のノードがあります。
- 自軍4人、敵複数のターン制戦闘。
- 味方は下級職に就き、在職中のみ能力補正と職業スキルを得ます。
- HP0は死亡ではなく「戦闘不能」。休憩所の復帰で戻せます。
- 各ユニットは所持品4枠を持ち、武器と消費アイテムを同じ欄で管理します。
- ラン全体で共有する容量無制限の輸送隊があり、ワールドマップの「身支度」で所持品を出し入れできます。
- 初期装備は青銅系武器 + 傷薬。青銅系は低威力・高耐久の詰み防止用です。
- 戦闘後は報酬候補3つから1つを選び、空き所持品欄のある味方か輸送隊へ送ります。
- 休憩所では、休息・復帰・鍛錬・修繕から2回行動できます。

## 操作

1. ワールドマップで「探索開始」または「次へ」を押す。「身支度」では輸送隊と味方の所持品を管理できます。
2. 自軍ユニットをクリックして選択する。
3. 青い範囲内の移動先をクリックする。
4. 右側メニューから「攻撃」「強撃」「装備変更」「道具」「待機」「やめる」を選ぶ。
5. 攻撃・強撃を選んだら、赤く表示された敵をクリックする。
6. 戦闘予測を確認し、「戦う」で確定する。
7. 戦闘終了後、報酬候補3つから1つを選び、持たせる味方か輸送隊を選ぶ。

身支度では、ユニットを選んで所持品を預ける・取り出す・装備変更・道具使用ができます。輸送隊は戦闘中には利用できません。

装備変更は行動権を消費しません。道具使用は行動権を消費します。

## 主要仕様の要約

- 強撃は1マップ1回。
- 強撃ダメージ = 通常ダメージ + 3 + max(攻撃側の技 - 防御側の技, 0)。
- 命中 = 武器命中 + 攻撃側の実効技 * 2 - 防御側の実効速さ * 2 + 職業スキル補正。
- 命中判定は2RN。
- 速さ差4以上で追撃。
- 反撃側も速ければ追撃反撃します。
- レベルアップ時は個人成長3ポイントに加え、現在の職業成長率から1ポイント上昇します。
- 同陣営ユニットは移動時に通過可能、停止不可です。
- 報酬レアリティは、コモン70%、アンコモン25%、レア5%。
- 報酬カテゴリは、武器60%、消費アイテム40%。カテゴリ保証はありません。

詳しい仕様は `docs/DESIGN.md` を参照してください。

## ディレクトリ構成

```txt
src/
  main.ts              # ゲーム状態、入力、進行、描画の接着
  types.ts             # 型定義
  constants.ts         # 画面サイズ、共通常数
  data/
    weapons.ts         # 武器テーブル
    items.ts           # 消費アイテムテーブル
    playerUnits.ts     # 味方ユニット初期値
    classes.ts         # 下級職、能力補正、職業スキル、職業成長率
    enemies.ts         # 敵ユニット基本値
    maps.ts            # 戦闘マップ、ワールドマップ
    rewards.ts         # 報酬レート、報酬テーブル
  logic/
    combat.ts          # 戦闘計算、戦闘予測
    inventory.ts       # 所持品、装備、空き枠処理
    classes.ts         # 職業参照、実効能力値
    enemyAI.ts         # 敵AI
    factories.ts       # マスターデータから実体データ生成
    rewards.ts         # 報酬抽選
  ui/
    theme.ts           # UI配色、文字階層
    canvas.ts          # ウインドウ、ボタン、ゲージなどの共通描画
    widgets.ts         # ログ、HP表示などの複合UI

docs/
  DESIGN.md            # 現行仕様と設計意図
  CODEX_GUIDE.md       # Codex向け作業ルール
  TODO.md              # 次に進めるタスク候補
  CHANGELOG.md         # 変更履歴
```

## Codexで作業するときの基本ルール

- 作業前に `docs/DESIGN.md` と `docs/CODEX_GUIDE.md` を読む。
- 作業後に必ず `npm run build` を実行する。
- 仕様変更をしたら、`README.md` または `docs/` の該当ファイルも更新する。
- データ追加は `src/data/` に置く。
- 戦闘計算は `src/logic/combat.ts`、敵AIは `src/logic/enemyAI.ts`、所持品処理は `src/logic/inventory.ts` に置く。
- `main.ts` に大きなロジックを戻さない。
